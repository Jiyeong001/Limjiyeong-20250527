# 올웨이즈 리뷰 진단 — 과제 제출

> 쇼핑몰 리뷰의 신뢰도를 진단하고, 바이럴을 걸러낸 신뢰도 높은 리뷰를 AI로 요약하는 MVP

**배포 URL:** https://always-production.up.railway.app

---

## 기술 스택

| 영역 | 스택 |
|---|---|
| 프론트엔드 | React + Vite (JavaScript) |
| 백엔드 | Node.js + Express (JavaScript) |
| 크롤링 | Puppeteer-extra + StealthPlugin |
| AI 요약 | OpenAI GPT-4o-mini |
| 배포 | Railway (단일 서버, React 빌드 정적 서빙) |

## 구현 설명

### 왜 올리브영인가

올웨이즈가 판매하는 건강기능식품 카테고리와 상품 구성이 가장 유사하고, 리뷰 볼륨이 충분한 플랫폼이 올리브영이었습니다. 특히 올리브영의 리뷰 데이터에는 **재구매 여부(`isRepurchase`), 장기 사용 여부(`isLongTermUse`), 우수 리뷰어 등급(`isTopReviewer`, `reviewerRank`), 도움이 돼요 수(`usefulPoint`), 체험단 여부(`isExperienceReview`)** 같은 신뢰도 판별에 직접 활용할 수 있는 메타데이터가 포함되어 있었습니다. 단순 별점/텍스트만 있는 플랫폼보다 신뢰도 스코어링의 정밀도를 높일 수 있어 선택했습니다.

### 리뷰 크롤링 방식

올리브영 리뷰 API(`POST https://m.oliveyoung.co.kr/review/api/v2/reviews/cursor`)는 브라우저 세션 쿠키가 없으면 빈 응답을 반환합니다. 이를 우회하기 위해 Puppeteer로 실제 상품 페이지를 먼저 로드해 세션을 확보한 뒤, **브라우저 컨텍스트 내부의 `fetch`로 API를 호출**하는 방식을 사용했습니다. 외부에서 직접 API를 호출하면 CORS + 세션 문제가 발생하지만, 브라우저 내부에서 호출하면 세션이 자동으로 포함되어 정상 응답을 받을 수 있습니다.

페이지네이션은 커서 기반으로, API 응답의 `nextCursorId`와 `nextCursorScore`를 다음 요청에 실어 최대 150개까지 수집합니다. `hasNext: false`가 되거나 커서가 변하지 않으면 중단합니다.

### 가격 크롤링

올리브영은 React CSS Modules를 사용해 클래스명에 해시가 붙습니다(`GoodsDetailInfo_price__AoTh8` 등). 배포 시마다 해시가 바뀔 수 있어 정확한 클래스 셀렉터를 사용할 수 없었습니다. 부분 매칭 셀렉터(`[class*="price__"]`, `[class*="price-before"]`)로 현재 판매가와 정가를 추출합니다. 12개 상품 전체의 메타를 상품 목록 진입 시 배치(3개씩 병렬)로 가져오고, 상품 상세 진입 시에도 재요청해 항상 최신 가격을 표시합니다.

### 리뷰 건수 처리

올리브영 리뷰 API 응답에는 `totalCount` 필드가 없습니다. 현재는 두 가지로 처리했습니다.

- **상품 목록**: `hasNext: false`이면 수집한 리스트 길이 = 실제 전체 건수이므로 표시. `hasNext: true`(리뷰 20개 초과)이면 null로 두어 UI에서 숨깁니다.
- **상품 상세**: 전체 분석 후 수집된 리뷰 수(`totalReviews`)를 "N개 분석" 형태로 표시합니다.

정확한 총 건수를 가져오려면 상품 상세 페이지 DOM의 별도 영역을 파싱하거나, 다른 API 엔드포인트를 탐색해야 하는데 이 부분은 아쉬움으로 남습니다.

### 신뢰도 스코어링

기본 점수 0.5에서 시작해 각 신호를 더하고 빼는 방식입니다.

| 신호 | 방향 | 근거 |
|---|---|---|
| `isExperienceReview` (체험단) | -0.4 | 보상을 받은 리뷰는 긍정 편향이 강함 |
| `isRepurchase` (재구매) | +0.2 | 실제 돈을 써서 다시 산다는 가장 강한 진성 신호 |
| `isLongTermUse` (장기 사용) | +0.15 | 충동적 반응이 아닌 사용 경험 기반 |
| `isTopReviewer` (우수 리뷰어) | +0.1~0.25 | 플랫폼이 공인한 리뷰어, 랭킹 반영 |
| `usefulPoint` (도움이 돼요) | +0.05~0.15 | 다른 구매자들의 커뮤니티 검증 |
| `hasPhoto` (사진 리뷰) | +0.05 | 조작 난이도가 높음 |
| 텍스트 길이·부정어 포함 | ±0.1~0.25 | 짧고 과도하게 긍정적인 리뷰 패널티 |
| 날짜 집중도 | -0.05~0.3 | 특정 날 리뷰가 몰리면 바이럴 의심 |

### AI 요약

신뢰도 상위 30개 리뷰를 GPT-4o-mini에 전달해 장점·단점·주의사항을 요약합니다. 바이럴성 리뷰보다 재구매·장기사용 신호가 강한 리뷰를 우선 인풋으로 사용하며, 유저의 페르소나 기반 관심사를 프롬프트에 반영해 **유저마다 다른 요약**을 생성합니다.

리뷰 크롤링 결과는 24시간 캐시하지만 요약은 캐시하지 않아, 재진입 시 크롤링은 스킵하면서도 개인화 요약은 항상 새로 생성합니다.

### 세션 기반 개인화

세션이 시작되면 타겟 페르소나(35–52세 여성, 주부·직장인·교사 등) 100명의 mock 유저 중 1명이 랜덤으로 할당됩니다. 세션 내내 이 페르소나는 고정되며, Jaccard 유사도로 해당 페르소나와 가장 유사한 peer group을 구성합니다. 두 번째 상품부터 peer group의 주요 관심사(면역력·피부·장건강 등)가 GPT 프롬프트에 반영되고 배너로 표시됩니다. 상품 클릭 히스토리는 페르소나와 별도로 누적됩니다.

실서비스에서는 랜덤 할당 대신 로그인 유저의 실제 구매 이력으로 페르소나를 결정하고, mock 유저 대신 올웨이즈 실 유저 DB를 활용하면 동일한 구조로 고도화 가능합니다.

## 올웨이즈 DB 연동 시 효과

현재 `client/src/data/products.js`에 올리브영 goodsNo 기반으로 12개 상품이 하드코딩되어 있습니다. 이 부분을 올웨이즈 실제 DB와 연동하면 다음과 같은 효과를 기대할 수 있습니다.

**즉각적인 효과**
- 올웨이즈 전체 상품 카탈로그에 리뷰 진단 기능 적용 가능
- 상품 정보(이름·가격·이미지)를 올영 크롤링 대신 DB에서 직접 가져와 신뢰도 향상
- 카테고리·브랜드 필터링 등 DB 기반 정교한 탐색 가능

**중기적 효과**
- 분석 결과를 DB에 캐싱하면 반복 요청 시 크롤링 없이 즉시 응답 가능 (현재는 상품 진입마다 재크롤링)
- 올웨이즈 유저의 클릭·구매 데이터와 리뷰 신뢰도 점수를 결합한 추천 품질 개선
- 리뷰 신뢰도 낮은 상품을 탐지해 MD 큐레이션이나 상품 노출 조정에 활용 가능

## 강조하고 싶은 부분

**별점보다 신뢰도** — 올리브영에서 별점 4.9점 상품들도 체험단 비율이 높으면 실구매 평균과 괴리가 있습니다. 이 MVP는 별점을 무시하지 않되, 신뢰도 기반으로 정렬된 리뷰와 필터링된 요약을 별도로 제공해 사용자가 더 정확한 판단을 내릴 수 있게 합니다.

**브라우저 세션 공유** — Puppeteer 브라우저 인스턴스를 싱글턴으로 유지해 매 요청마다 Chrome을 재시작하는 오버헤드를 제거했습니다. 12개 상품 메타를 한 번에 배치 요청해도 브라우저 재시작 없이 처리됩니다.

**개인화 구조의 확장성** — 현재는 mock 유저 기반이지만, 세션 할당 → peer group → GPT 프롬프트 반영으로 이어지는 파이프라인은 실 데이터 연동 시 구조 변경 없이 고도화됩니다. 요약 캐시를 분리한 것도 이 확장을 염두에 둔 설계입니다.

## 확장 기능 (로드맵)

### 초개인화 요약 (로그인 연동 후)
현재는 세션 시작 시 랜덤 페르소나를 할당해 peer group을 구성합니다. 실서비스에서는 소셜/이메일 로그인 후 유저의 실제 구매 이력으로 페르소나를 결정하고, 재방문 시 '지난번에 건기식을 많이 보셨네요' 형태의 장기 개인화가 가능합니다. 올웨이즈가 이미 보유한 유저 구매 데이터와 연동하면 추가 데이터 수집 없이 즉시 고도화 가능합니다.

### 유사 페르소나 기반 리뷰 가중치 조정
현재 peer group은 GPT 프롬프트 힌트로만 사용되지만, trust score 계산 시 peer group의 공통 관심사(예: 피부)와 연관된 리뷰 키워드에 가중치를 부여하면 신뢰도 정렬 자체도 개인화할 수 있습니다. 올웨이즈 실 유저의 카테고리 친화도 데이터가 있다면 가중치 튜닝도 데이터 기반으로 가능합니다.

## 아쉬운 점 / 개선 여지

- **리뷰 크롤링 캐싱, 요약은 미캐시**: 크롤링 결과는 24시간 파일 캐시로 재사용하지만, 개인화 요약은 매번 GPT를 호출합니다. 트래픽이 많아지면 유사 페르소나 간 요약을 공유하거나 Redis로 교체해야 합니다.
- **리뷰 총 건수 미표시**: 올리브영 API에 totalCount 필드가 없어 정확한 전체 건수를 표시할 수 없습니다.
- **GPT 스트리밍 미적용**: 현재 요약 생성이 완료될 때까지 로딩이 유지됩니다. Streaming API를 적용하면 응답이 생성되는 즉시 사용자에게 표시할 수 있습니다.
- **신뢰도 스코어 튜닝**: 현재 가중치는 논리적 근거 기반이지만 실제 구매 전환 데이터와 교차 검증하지 못했습니다. 올웨이즈 실 데이터가 있다면 A/B 테스트로 최적 가중치를 찾을 수 있습니다.
- **상품 범위**: 올리브영 상품 12개로 제한되어 있으며, 올웨이즈 DB 연동 전까지는 수동으로 goodsNo를 추가해야 합니다.
